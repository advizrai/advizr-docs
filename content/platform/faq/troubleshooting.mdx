# Troubleshooting

Solutions to common issues you may encounter on the platform. Each section describes the problem, the likely cause, and the steps to fix it. If none of these solve your issue, contact your FDE through [Chat](/docs/platform/dashboard/chat) or [book a support call](/docs/platform/dashboard/book).

<Details title="I cannot log in">
  **Likely cause:** Incorrect password or expired reset link.

  1. Double-check that you are using the correct email address
  2. Click "Forgot password" on the login page and follow the reset steps
  3. Check your spam folder for the reset email
  4. Make sure you are visiting advizrclients.com (not a different URL)
  5. If you still cannot log in, contact your Advizr team at support for a manual reset
</Details>

<Details title="The page is not loading or loads slowly">
  **Likely cause:** Browser cache, network issue, or outdated browser.

  1. Refresh the page (Ctrl+R or Cmd+R)
  2. Clear your browser cache and cookies for advizrclients.com
  3. Try a different browser (Chrome, Firefox, Safari, or Edge)
  4. Check your internet connection
  5. If the issue persists, the platform may be undergoing maintenance - wait a few minutes and try again
</Details>

<Details title="Chat is not responding">
  **Likely cause:** Temporary processing delay or session timeout.

  1. Wait 10-15 seconds - some responses take time, especially for document analysis
  2. Check if a loading indicator is visible (the AI may still be processing)
  3. Refresh the page and try sending your message again
  4. Start a new thread and retype your question
  5. If Chat remains unresponsive, contact your FDE through email or book a call
</Details>

<Details title="Document upload is failing">
  **Likely cause:** File too large, unsupported format, or network interruption.

  1. Check the file size - maximum is 25 MB per file
  2. Check the file format - supported types are PDF, DOCX, TXT, XLSX, CSV, PNG, JPG, and JPEG
  3. Try uploading a smaller file to confirm the upload function works
  4. If uploading over a slow connection, try again on a faster network
  5. If the problem persists, try a different browser
</Details>

<Callout type="warning">
  Before trying other fixes, clear your browser cache. Many display and loading issues are caused by stale cached files. In Chrome, go to Settings, Privacy and security, Clear browsing data.
</Callout>

<Details title="Notifications are not arriving">
  **Likely cause:** Email digest is turned off, or emails are going to spam.

  1. Check your [notification preferences](/docs/platform/features/notifications) to make sure email digests are enabled
  2. Check your spam or junk folder for emails from Advizr
  3. Add the Advizr sender address to your email contacts or safe senders list
  4. In-app notifications should still appear even if email is disabled - check the bell icon in the top bar
</Details>

<Details title="The display looks wrong on mobile">
  **Likely cause:** Screen size or zoom level.

  1. Make sure your browser zoom is set to 100%
  2. Try rotating your phone to landscape mode
  3. If you added the site to your home screen, remove it and re-add it
  4. Update your browser to the latest version
  5. The platform is optimised for screens 320px and wider - very small screens may have layout limitations
</Details>

<Details title="My session keeps expiring">
  **Likely cause:** The 24-hour session token has expired.

  Sessions last 24 hours and renew automatically while you are active. If you leave the platform idle for an extended period, you will need to log in again. This is a security feature, not a bug. See [Login and Security](/docs/platform/account/login-and-security) for more details.
</Details>

<Details title="How do I report a bug?">
  If you have found a bug or something that does not look right:

  1. Take a screenshot of the issue
  2. Note what you were doing when it happened
  3. Open [Chat](/docs/platform/dashboard/chat) and describe the issue with the screenshot attached
  4. Your FDE will investigate and follow up

  For urgent issues that prevent you from using the platform, email your FDE directly or book an ad-hoc call.
</Details>
