# Monitor runs and workflows

Your Advizr team builds automated workflows that handle tasks for your business — things like generating reports, processing documents, analyzing data, and executing multi-step processes. Each time a workflow executes, it creates a "run" that you can monitor and review. The Runs view shows you what is running, what has completed, and what the results are.

## Where to find Runs

Runs does not appear in the main sidebar (Home, Chat, Goals, Documents, Proposals, Learn, Book a Call, Help). It is part of Advanced mode, which you turn on in Settings. {/* GATE0: confirm Runs lives under Settings -> Advanced mode */}

If you have never switched modes, see [Configure your settings](/docs/platform/features/settings) for how Simple and Advanced mode work.

## What workflows are

Workflows are automated sequences that your FDE builds to address your business objectives. For example:

- A workflow that generates a monthly financial summary from your uploaded data
- A workflow that processes incoming leads and enriches them with public data
- A workflow that analyzes customer feedback and produces a trends report
- A workflow that drafts email sequences based on your brand guidelines

Each workflow is configured specifically for your business using the documents and context you have provided.

<Details title="Why your Advizr team builds workflows for you">
  Workflows come out of your objectives and approved proposals, and they depend on correct configuration — data sources, schedules, error handling. Your FDE owns that setup so the automation is reliable from day one. You never need to configure a workflow yourself; your job is to review the outputs and tell your team what to adjust.
</Details>

## How runs are triggered

A run is a single execution of a workflow. Runs start in three ways:

- **By your FDE** - Your team member starts a run manually as part of their work
- **By you via Chat** - Ask in Chat to run a specific workflow and it can be triggered directly
- **On a schedule** - Some workflows run automatically at set intervals (daily, weekly, monthly)

## Monitor a run

<Steps>
  <Step title="Turn on Advanced mode">
    Open **Settings** and switch the interface to Advanced mode so the Runs view is visible. {/* GATE0: confirm exact toggle label and location with James */}
  </Step>
  <Step title="Open the Runs view">
    Open Runs from the navigation, or follow the link in a run notification. You will see a list of recent runs with their status.
  </Step>
  <Step title="Check the status">
    Each run shows one of four statuses: **Triggered** (queued to start), **Running** (in progress), **Completed** (finished successfully), or **Failed** (something went wrong).
  </Step>
  <Step title="View the details">
    Click a run to see its step-by-step log. Each step shows what happened, how long it took, and whether it succeeded. For completed runs, the output is available to download or review.
  </Step>
</Steps>

You should now see the run's detail view with its step log, and for completed runs, the output ready to download or review.

## Review outputs

When a run completes, its output appears in the run detail view. Depending on the workflow, outputs might include generated documents, data tables, analysis summaries, or status reports. You can download outputs or reference them in Chat.

## If something goes wrong

- **You cannot find the Runs view.** Check that Advanced mode is turned on in [Settings](/docs/platform/features/settings) — Runs is hidden in Simple mode.
- **A run shows Failed.** Open the run to see which step failed. Your FDE is automatically notified of failed runs and will investigate — you do not need to take action, but you can ask for an update in Chat.
- **A run sits in Triggered or Running for a long time.** Some workflows take a while, especially document-heavy ones. If nothing changes after an hour, mention the run in Chat.
- **An output is missing or looks wrong.** Tell your FDE in Chat which run and which output — they can re-run the workflow or correct the configuration.

Still stuck? Send your FDE a message in Chat with your browser, a screenshot, and when it happened.
